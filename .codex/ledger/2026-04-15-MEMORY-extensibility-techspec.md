  - HTTP `/api/resources` mutations stay disabled until explicit operator auth middleware exists

  - bundle ownership fan-out is explicitly separate from projector dependency edges
  - projector atomicity is defined differently for data projectors versus connection-backed projectors
- Typed-boundary refinement:
  - raw JSON remains confined to an internal `rawStore`, `RawDraft`, and `RawRecord`
  - domain-facing contracts now use typed `Store[T]`, `Draft[T]`, `Record[T]`, `KindCodec[T]`, and `TypedProjector[T]`
  - `Store[T]` read methods intentionally keep `MutationActor` so same-source and same-scope filtering stays enforced at the typed façade edge
  - mixed-kind outliers such as `bundle.activation` are explicitly handled by package-local `Projector` adapters instead of over-generalizing `TypedProjector[T]`

  - the sandbox provider core remains compatible with extensibility-parity
  - the sandbox spec must be revised where it crosses the new extension runtime: hook lifecycle exposure, extension Host API capability wiring, and daemon boot ordering
  - `environment/*` remains an operational API family, not a new desired-state resource family
  - sandbox implementation should split into two tracks: provider/session/workspace/config can proceed independently, while environment hooks and extension-facing Host API should align with extensibility tasks 02-07 semantics before landing
