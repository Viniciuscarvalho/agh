# ADR-004: Do Not Implicitly Inherit Parent Soul in Spawned Sessions

## Status

Accepted

## Date

2026-05-01

## Context

AGH supports spawned sessions with explicit lineage, spawn role, prompt overlays, permissions, and narrowing. Persona inheritance can contaminate specialized roles: a coordinator persona should not silently shape a reviewer, implementer, QA agent, or external/networked agent. Hermes isolates delegated subagents by default and uses focused prompts rather than inheriting parent persona.

## Decision

Spawned sessions will not implicitly inherit the parent session's `SOUL.md`. A spawned session uses the target agent's own resolved `SOUL.md`, if present, plus explicit spawn overlays such as `spawn_role` and `prompt_overlay`.

Parent soul may appear only as compact lineage/provenance metadata, not as behavioral instruction. MVP stores this as `parent_soul_digest` on the spawned session record and exposes it only through lineage/read-model payloads. Any future partial inheritance must be explicit and typed, not the default.

## Alternatives Considered

### Alternative 1: Inherit Parent Soul by Default

- **Description**: Child sessions receive the parent's full persona unless overridden.
- **Pros**: Preserves a house style for child work.
- **Cons**: Role contamination and unclear authority boundaries.
- **Why rejected**: Spawn already has explicit overlays; implicit inheritance is too surprising.

### Alternative 2: Never Expose Parent Soul

- **Description**: Omit all parent soul information from children.
- **Pros**: Maximum isolation.
- **Cons**: Loses useful audit lineage.
- **Why rejected**: Provenance is useful as metadata, as long as it is not behavioral instruction.

## Consequences

### Positive

- Keeps spawned roles clean and predictable.
- Preserves target-agent identity semantics.
- Avoids accidental policy/persona leakage.

### Negative

- Authors must explicitly pass shared communication guidance through overlays or agent definitions.

### Risks

- Operators may expect child agents to share parent style; docs and spawn examples must make this explicit.

## Implementation Notes

- Resolve target agent soul at spawn/session creation.
- Store `parent_soul_digest` only as lineage/provenance metadata.
- Keep existing `AgentSpawnRequest.SpawnRole`, `AgentSpawnRequest.PromptOverlay`, `session.SpawnOpts.SpawnRole`, and `session.SpawnOpts.PromptOverlay` as explicit ephemeral context.
